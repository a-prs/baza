# Что дальше: твой путь вайб-кодера

Ты освоил базу: поднял сервер, задеплоил проект, настроил автоматизации. Что делать дальше — зависит от того, куда хочешь расти. Разберём несколько направлений.

## Три пути

### Путь 1: Строю больше продуктов

Ты хочешь быстро запускать новые проекты. Тогда сфокусируйся на:

- **Типовые шаблоны** — у тебя уже есть проверенные стеки (FastAPI + SQLite, aiogram + aiosqlite). Кодируй их как шаблоны, чтобы каждый новый проект начинался с готовой базы.
- **CLAUDE.md в каждом проекте** — файл-инструкция для ИИ специфичный к проекту. Опиши стек, соглашения по коду, что нельзя менять. ИИ будет работать точнее.
- **Деплой автоматизированно** — настрой GitHub Actions чтобы при пуше в main код сам ехал на сервер.

### Путь 2: Углубляюсь в ИИ-инструменты

Тебе интересно как работают MCP, агенты, многошаговые задачи:

- **MCP-серверы** — подключай новые инструменты к Claude: базы данных, API сервисов, файловую систему. Библиотека на [github.com/modelcontextprotocol](https://github.com/modelcontextprotocol/servers).
- **Claude Projects** — создай постоянный проект в Claude.ai с инструкциями и контекстом. Больше не надо объяснять ИИ каждый раз что за проект.
- **Агентные workflow** — цепочки задач: ИИ сам планирует шаги и выполняет их последовательно. Начни с n8n + ИИ-нодами.

### Путь 3: Монетизирую навык

Ты хочешь зарабатывать на том что умеешь:

- **Фриланс** — внедрение автоматизаций, Telegram-боты, RSS-дайджесты для клиентов. Стандартный проект: 20-80 тысяч рублей.
- **SaaS-инструменты** — запусти подписочный сервис для нишевой аудитории. Вайб-кодинг позволяет сделать MVP за неделю.
- **Контент** — блог, YouTube, Telegram-канал о том как ты создаёшь проекты. Аудитории интересен живой процесс.

## Следующие технические шаги

### Автоматический деплой через GitHub Actions

```yaml
# .github/workflows/deploy.yml
name: Deploy

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Deploy to server
        uses: appleboy/ssh-action@v0.1.10
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USER }}
          key: ${{ secrets.SSH_PRIVATE_KEY }}
          script: |
            cd /opt/my-app
            git pull
            source .venv/bin/activate
            pip install -r requirements.txt
            systemctl restart my-app
```

Добавь секреты в GitHub (Settings → Secrets): `SERVER_HOST`, `SERVER_USER`, `SSH_PRIVATE_KEY`. После этого каждый `git push` автоматически деплоит на сервер.

### Docker — одинаковая среда везде

Docker решает проблему «на моей машине работало»: упаковываешь приложение с зависимостями в контейнер.

```dockerfile
# Dockerfile
FROM python:3.11-slim

WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt

COPY . .
CMD ["python", "bot.py"]
```

```bash
# Запустить
docker build -t my-bot .
docker run -d --restart=always --env-file .env --name my-bot my-bot
```

Попроси ИИ:

```
Создай Dockerfile для моего Python-приложения.
Файл точки входа: bot.py
Зависимости: в requirements.txt
Переменные окружения: через .env файл при запуске
```

### WebSocket — живые обновления

Если хочешь дашборд с живыми данными (без перезагрузки страницы):

```
Добавь WebSocket в мой FastAPI:
- подключение клиентов к /ws
- каждые 5 секунд отправлять обновлённые данные из базы
- во фронтенде показывать без перезагрузки
```

### Telegram Mini Apps

Веб-приложение прямо внутри Telegram — пользователи не выходят из мессенджера:

```
Создай Telegram Mini App:
- простая форма для заполнения заявки
- при отправке — запись в базу и уведомление мне
- стили адаптивные под мобильный
```

## Чеклист зрелого вайб-кодера

- [ ] Каждый проект имеет `CLAUDE.md` с описанием стека
- [ ] Секреты в `.env`, в `.gitignore`, никогда в коде
- [ ] Сервисы на systemd с `Restart=always`
- [ ] Базовый мониторинг: уведомления при падении
- [ ] Git с осмысленными коммитами
- [ ] Деплой не требует ручной работы (автоматически или одной командой)
- [ ] README.md в каждом репозитории (попроси ИИ написать)

## Главный принцип

Вайб-кодинг — это не про знание синтаксиса. Это про умение объяснить задачу и проверить результат.

**Чем точнее объясняешь — тем меньше правишь.**
**Чем больше запускаешь — тем лучше понимаешь.**

Лучший следующий шаг — взять конкретную задачу из своей жизни или работы и попробовать автоматизировать её с помощью ИИ. Не обязательно идеально — главное запустить.
