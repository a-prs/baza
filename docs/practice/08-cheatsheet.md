## Глоссарий

| Термин | Что это |
|--------|---------|
| **API** | Способ, которым программы общаются друг с другом |
| **CLI** | Программа, которая работает через терминал (текстовые команды) |
| **CLAUDE.md** | Файл с правилами и контекстом для ИИ |
| **CODE_MAP.md** | Карта проекта — какой файл за что отвечает |
| **Деплой** | Выложить проект на сервер, чтобы он был доступен в интернете |
| **DNS** | Система, которая переводит домен (имя) в IP-адрес (числа) |
| **Домен** | Красивый адрес сайта (например, mysite.com) |
| **Git** | Система контроля версий — «машина времени» для файлов |
| **GitHub** | Онлайн-хранилище Git-проектов |
| **HTTPS/SSL** | Защищённое соединение (замочек в браузере) |
| **IDE** | Программа для написания и редактирования кода (например, VS Code) |
| **IP-адрес** | Числовой адрес сервера в интернете |
| **Low-code** | Создание проектов с минимумом кода (n8n, Retool) |
| **MCP** | Протокол подключения ИИ к внешним сервисам |
| **No-code** | Создание проектов без кода (Tilda, Notion) |
| **Node.js** | Среда для запуска JavaScript вне браузера |
| **n8n** | Платформа автоматизаций (связывает сервисы в цепочки) |
| **SSH** | Удалённое подключение к серверу через терминал |
| **Терминал** | Окно для текстовых команд компьютеру |
| **VPS** | Виртуальный сервер, который ты арендуешь |
| **VS Code** | Бесплатная IDE от Microsoft — «швейцарский нож» кодера |

## Частые ошибки новичков

### «Команда не найдена» (command not found)
**Причина:** программа не установлена или не добавлена в PATH.
**Решение:** переустанови программу, убедись что поставил галочку «Add to PATH». После установки — перезапусти VS Code.

### «Нет прав» (permission denied)
**Причина:** нужны права администратора.
**Решение:** на сервере — добавь `sudo` перед командой. На Windows — запусти терминал от имени администратора.

### «Порт занят» (port already in use / EADDRINUSE)
**Причина:** другая программа уже использует этот порт.
**Решение:** найди и останови процесс:
```bash
lsof -i :8080    # найти что занимает порт 8080
kill -9 <PID>    # остановить по ID
```
Или просто перезапусти сервер / смени порт в настройках.

### «Соединение отклонено» (connection refused / ECONNREFUSED)
**Причина:** приложение на этом порту не запущено.
**Решение:** проверь, запущен ли сервис:
```bash
systemctl status имя-сервиса
```
Если не запущен — `systemctl start имя-сервиса`.

### «502 Bad Gateway»
**Причина:** Nginx запущен, но приложение, к которому он проксирует, не работает.
**Решение:** проверь статус приложения и его логи:
```bash
systemctl status имя-приложения
journalctl -u имя-приложения -n 50
```

### «Не найден модуль» (ModuleNotFoundError / Cannot find module)
**Причина:** библиотека не установлена, или установлена не в то виртуальное окружение.
**Решение:**
```bash
pip install имя-пакета      # Python
npm install имя-пакета      # Node.js
```
Если используешь venv — убедись что он активирован (`source .venv/bin/activate`).

### DNS не обновился
**Причина:** DNS-записи обновляются не мгновенно — от 5 минут до нескольких часов.
**Решение:** подожди. Проверить текущий DNS можно командой:
```bash
nslookup mysite.com
```
Или через сервис [dnschecker.org](https://dnschecker.org).

### SSH — «Connection refused» на порту 22
**Причина:** сервер недоступен, или SSH не запущен.
**Решение:** попробуй зайти через консоль хостинга (у Hetzner/DO есть веб-консоль). Проверь что SSH работает: `systemctl status ssh`.

### ИИ делает не то, что я прошу
**Причина:** промпт слишком размытый.
**Решение:** будь конкретнее. Вместо «сделай красиво» → «сделай тёмный фон #1a1a2e, белый текст, шрифт Inter 16px, кнопки со скруглением 8px».

### ИИ сломал то, что работало
**Причина:** ИИ внёс слишком много изменений за раз.
**Решение:** используй Git (`git diff` — посмотреть что изменилось, `git checkout .` — откатить все изменения). Давай ИИ задачи маленькими порциями.

### .env не читается (переменная = None)
**Причина:** `load_dotenv()` не вызван, или `.env` лежит не в той папке.
**Решение:** убедись что `load_dotenv()` вызывается в начале файла, и что `.env` лежит в корне проекта (там же где запускаешь скрипт).

## Команды Git (шпаргалка)

```bash
git status                    # что изменилось
git diff                      # посмотреть изменения
git add .                     # добавить всё
git commit -m "описание"      # сохранить
git log --oneline             # история
git checkout .                # откатить все несохранённые изменения
git checkout -b имя-ветки     # создать ветку
git push                      # отправить на GitHub
git pull                      # забрать с GitHub
```

## Команды сервера (шпаргалка)

```bash
systemctl status имя          # статус сервиса
systemctl start имя           # запустить
systemctl stop имя            # остановить
systemctl restart имя         # перезапустить
journalctl -u имя -f          # логи в реальном времени
ss -tlnp                      # что слушает на каких портах
nginx -t                      # проверить конфиг nginx
systemctl reload nginx        # перезагрузить nginx без downtime
```
