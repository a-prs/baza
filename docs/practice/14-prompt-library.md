# Библиотека промптов

Вайб-кодинг — это разговор с ИИ. Чем точнее формулировка, тем лучше результат. Здесь собраны готовые шаблоны для самых частых задач. Копируй, меняй под себя, используй.

## Как пользоваться шаблонами

Слова в `[квадратных скобках]` — заменяй на своё. Остальное можно оставить как есть или адаптировать.

## Создание проектов с нуля

### Сайт-визитка

```
Создай одностраничный сайт-визитку.

Дизайн:
- Тёмный фон: #[цвет], текст: #[цвет]
- Акцентный цвет: #[цвет]
- Шрифт: системный sans-serif

Содержание:
- По центру: имя "[ИМЯ]", подпись "[ДОЛЖНОСТЬ/СЛОГАН]"
- [N] кнопок-ссылок: [НАЗВАНИЕ] (ссылка: [URL])
- Внизу копирайт: "© [ГОД] [ИМЯ]"

Требования:
- Всё в одном файле index.html
- Адаптивно под мобильные
- Плавные анимации при наведении
```

### Telegram-бот

```
Создай Telegram-бота на aiogram 3 (Python).

Команды:
- /start — [описание приветствия]
- /help — список команд
- [другие команды]

Логика:
- [описание что делает бот]

Требования:
- Токен берёт из .env переменной BOT_TOKEN
- Логи в консоль
- Обработка ошибок с внятными сообщениями

Файлы: bot.py, .env.example
```

### Веб-приложение

```
Создай веб-приложение: [описание что делает].

Стек:
- Frontend: HTML + CSS + JavaScript (без фреймворков)
- Backend: Python FastAPI [или: Node.js Express]
- База данных: SQLite [или: без БД]

Функциональность:
- [фича 1]
- [фича 2]
- [фича 3]

Запуск: python app.py [или: node server.js]
```

### Скрипт-автоматизация

```
Напиши Python-скрипт, который [описание задачи].

Входные данные: [откуда берёт данные]
Что делает: [шаги]
Результат: [что выдаёт]

Запускать: python script.py
Расписание: [каждый день в 9:00 / при запуске / вручную]
```

## Правки и улучшения

### Изменить дизайн

```
Внеси изменения в дизайн:
1. [изменение 1]
2. [изменение 2]
3. [изменение 3]

Не трогай логику — только визуальная часть.
```

### Добавить функцию

```
Добавь в [файл/компонент] следующую функциональность:
[описание что нужно добавить]

Не меняй то, что уже работает.
```

### Сделать адаптивным

```
Сделай этот сайт адаптивным под мобильные устройства.
- На экранах меньше 768px: [описание как должно выглядеть]
- Навигация на мобильных: [бургер-меню / скрыть / стек]
- Кнопки на мобильных: полная ширина
```

### Оптимизировать скорость

```
Оптимизируй этот сайт для скорости загрузки:
- Убери неиспользуемый CSS
- Добавь lazy loading для изображений
- Минимизируй JavaScript
```

## Интеграции с сервисами

### Подключить Telegram-уведомления

```
Добавь отправку уведомлений в Telegram при [событие].

Параметры (берутся из .env):
- TELEGRAM_BOT_TOKEN — токен бота
- TELEGRAM_CHAT_ID — куда отправлять

Сообщение: "[шаблон сообщения, например: 🔔 Новая заявка от {name}: {message}]"
```

### Подключить базу данных

```
Добавь SQLite базу данных в проект.

Таблицы:
- [название таблицы]: [поля и их типы]

Операции:
- Сохранять [что] при [когда]
- Читать [что] для [зачем]

Используй aiosqlite [или: sqlite3]. Файл БД: data/app.db
```

### Подключить внешний API

```
Подключи API [название сервиса].
Документация: [ссылка или описание]
API ключ берётся из .env переменной: [НАЗВАНИЕ_КЛЮЧА]

Нужно:
- [что именно делать с API]
- Обработать ошибки (недоступен, неверный ключ, лимит)
```

## Советы по промптам

**Больше конкретики — лучше результат:**

❌ Плохо: `Сделай красиво`  
✅ Хорошо: `Фон #1a1a2e, кнопки скруглённые 8px, при наведении фиолетовая обводка #7c3aed`

**Один промпт — одна задача:**

❌ Плохо: `Добавь авторизацию, корзину, оплату и личный кабинет`  
✅ Хорошо: `Добавь форму входа: email + пароль, кнопка "Войти"`

**Описывай результат, не процесс:**

❌ Плохо: `Используй flexbox с gap 16px и justify-content center`  
✅ Хорошо: `Три карточки в ряд с одинаковыми промежутками, по центру страницы`

**Давай контекст при ошибке:**

❌ Плохо: `Не работает`  
✅ Хорошо: `При нажатии кнопки "Отправить" — страница перезагружается, форма не отправляется. Вот ошибка в консоли: [текст]`
