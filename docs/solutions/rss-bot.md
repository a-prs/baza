# RSS-бот в Telegram

Автоматический дайджест из любых RSS-источников прямо в Telegram. Бот собирает новые статьи, форматирует и присылает по расписанию. Настройка через n8n — без кода, только через интерфейс.

## Что получишь

- Telegram-сообщение раз в день (или раз в час — как настроишь)
- Статьи из любых RSS-источников: новости, блоги, YouTube-каналы
- Форматирование: заголовок + ссылка + краткое описание
- Дедупликация — одна статья не придёт дважды

**Пример:** каждое утро в 9:00 получаешь 5 свежих статей по ИИ с краткими описаниями.

## Шаг 1: Создай Telegram-бота

1. Открой [@BotFather](https://t.me/BotFather)
2. `/newbot` → имя → username
3. Скопируй **токен бота**

Узнай свой Chat ID — куда присылать:
1. Напиши что-нибудь своему боту
2. Открой в браузере: `https://api.telegram.org/bot<ТОКЕН>/getUpdates`
3. В ответе найди `"chat":{"id":XXXXXXX}` — это твой Chat ID

## Шаг 2: Найди RSS-ленты

Почти у каждого сайта есть RSS. Распространённые паттерны:

```
https://site.com/rss
https://site.com/feed
https://site.com/rss.xml
```

**Популярные RSS для начала:**
- TechCrunch: `https://techcrunch.com/feed/`
- Hacker News (топ): `https://news.ycombinator.com/rss`
- YouTube-канал: `https://www.youtube.com/feeds/videos.xml?channel_id=CHANNEL_ID`

## Шаг 3: Создай workflow в n8n

Открой n8n (`https://n8n.твойдомен.com`) и создай новый workflow.

### Структура workflow

```
[Schedule Trigger] → [RSS Feed Read] → [Filter: только новые] → [Telegram]
```

### Добавь триггер по расписанию

1. Нажми **+** → **Schedule Trigger**
2. Настрой: `Every Day at 09:00` (или как хочешь)

### Добавь чтение RSS

1. Нажми **+** → найди **RSS Feed Read**
2. **Feed URL:** вставь ссылку на RSS
3. В поле **Limit** поставь `10` (последние 10 статей)

### Фильтр новых статей

Чтобы одна статья не приходила дважды — используем n8n встроенный **Deduplication** (или **If** + **Set** для хранения просмотренных):

1. **+** → **Deduplicate Items** (или найди через поиск)
2. Поле для дедупликации: <code v-pre>{{ $json.link }}</code>

### Отправка в Telegram

1. **+** → **Telegram**
2. Нажми **Add Credential** → вставь токен бота
3. **Chat ID:** твой Chat ID из Шага 1
4. **Text:**

```text
📰 {{ $json.title }}

{{ $json.contentSnippet | truncate(200) }}

🔗 {{ $json.link }}
```

5. **Parse Mode:** `Markdown`

### Сохрани и активируй

Нажми **Save** → переключи тумблер **Active** в верхнем правом углу.

Готово — workflow будет запускаться по расписанию.

## Несколько источников

Хочешь собирать из 3-5 источников в одно сообщение? Добавь несколько RSS-нод и объедини их через **Merge**:

```
[RSS 1] ↘
[RSS 2] → [Merge] → [Deduplicate] → [Sort by date] → [Telegram]
[RSS 3] ↗
```

В ноде **Merge**: режим `Append` — просто объединяет все статьи в один список.

## Улучшения через ИИ

Если подключён MCP для n8n — можешь попросить ИИ доработать workflow:

```
Добавь в мой RSS-workflow ноду OpenAI:
она должна суммаризировать каждую статью в 2 предложения
на русском языке, и подставлять это как описание вместо contentSnippet
```

Или через n8n вручную: добавь ноду **OpenAI** между RSS и Telegram, с промптом:

```text
Суммаризируй эту статью в 2 предложения на русском:
{{ $json.content }}
```

## Управление workflow

```bash
# Посмотреть логи n8n (если что-то не работает)
docker compose -f /opt/n8n/docker-compose.yml logs -f
```

В интерфейсе n8n: вкладка **Executions** — история всех запусков, можно посмотреть что произошло при каждом.
