# Продвинутый промпт-инжиниринг

Базовый промпт — «сделай X». Продвинутый — структура которая надёжно даёт нужный результат. Разберём техники которые реально работают.

## Почему это важно для вайб-кодера

Ты общаешься с моделью не только в чате — ещё в системных промптах для своих продуктов. Плохой промпт = плохой продукт. Хороший промпт = конкурентное преимущество которое сложно скопировать.

## Структура хорошего промпта

```
[Роль] + [Контекст] + [Задача] + [Формат] + [Ограничения]
```

Плохо:
```
Напиши анализ рынка
```

Хорошо:
```
Ты — аналитик венчурного рынка с 10-летним опытом.

Контекст: стартап разрабатывает AI-ассистента для юристов. 
Целевые клиенты: малые юридические фирмы (2-10 человек) в РФ.

Задача: проанализируй рынок по фреймворку PEST (политика, экономика, социум, технологии).

Формат: 4 блока, каждый — 3-4 тезиса с конкретными данными/примерами.
Объём: не более 500 слов.

Ограничения: только факты, не общие слова. Если данных нет — честно скажи «данных нет».
```

## Техника Chain of Thought (CoT)

Попроси модель думать вслух перед ответом. Улучшает качество особенно для задач с рассуждением:

```
Реши задачу шаг за шагом:
1. Сначала опиши что известно
2. Выдели ключевые ограничения
3. Рассмотри несколько подходов
4. Выбери лучший и объясни почему
5. Дай финальный ответ

Задача: [твоя задача]
```

Или просто:
```
Думай шаг за шагом перед тем как дать ответ.
```

Это работает: модель находит больше ошибок в своём рассуждении когда выписывает его явно.


## Role Prompting

Задание роли меняет стиль и глубину ответа:

```
# Вместо:
Объясни что такое RAG

# Попробуй:
Ты — Senior ML Engineer объясняющий концепцию джуниор-разработчику.
Объясни RAG: начни с аналогии из повседневной жизни, потом технические детали.

# Или:
Ты — продавец AI-услуг. Объясни бизнесмену (не технарю) почему RAG важен
для его бизнеса. Используй примеры с деньгами и временем.
```

## Structured Output: JSON по требованию

Принудительно задать формат вывода:

```
Проанализируй текст и верни результат СТРОГО в JSON без markdown-обёртки:

{
  "sentiment": "позитивный|нейтральный|негативный",
  "confidence": 0.0-1.0,
  "key_phrases": ["фраза1", "фраза2"],
  "summary": "одно предложение"
}

Текст: "[текст для анализа]"

JSON:
```

Последнее слово «JSON:» заставляет модель начать с `{`.

## Промпт для последовательной работы

Для сложных задач — разбить на этапы:

```python
async def analyze_and_improve(text: str) -> dict:
    # Этап 1: анализ
    analysis = await claude.messages.create(
        model="claude-sonnet-4-6",
        messages=[{
            "role": "user",
            "content": f"""Проанализируй текст и выдели:
1. Главные проблемы (максимум 3)
2. Сильные стороны
3. Целевую аудиторию

Текст: {text}"""
        }]
    )
    
    # Этап 2: улучшение на основе анализа
    improved = await claude.messages.create(
        model="claude-sonnet-4-6",
        messages=[
            {"role": "user", "content": f"Вот текст для улучшения:\n{text}"},
            {"role": "assistant", "content": analysis.content[0].text},
            {"role": "user", "content": "На основе этого анализа перепиши текст. Устрани проблемы, сохрани сильные стороны."}
        ]
    )
    
    return {"analysis": analysis.content[0].text, "improved": improved.content[0].text}
```

## Итеративное уточнение

Когда нужно много попыток — встрой цикл улучшений:

```python
async def iterative_improve(content: str, criteria: str, max_iterations: int = 3) -> str:
    """Улучшать контент пока не пройдёт проверку"""
    current = content
    
    for i in range(max_iterations):
        # Проверить текущую версию
        check = await claude.messages.create(
            model="claude-haiku-4-5-20251001",
            messages=[{
                "role": "user",
                "content": f"""Проверь текст по критериям: {criteria}
                
Текст: {current}

Ответь JSON: {{"pass": true/false, "issues": ["проблема1", ...]}}"""
            }]
        )
        
        import json
        result = json.loads(check.content[0].text)
        
        if result["pass"]:
            break
        
        # Улучшить
        improve = await claude.messages.create(
            model="claude-sonnet-4-6",
            messages=[{
                "role": "user",
                "content": f"""Улучши текст устранив эти проблемы: {result['issues']}

Текст: {current}"""
            }]
        )
        current = improve.content[0].text
    
    return current
```

## Системный промпт vs пользовательский

```python
# system — контекст и правила, не меняется
# user — конкретная задача

await client.messages.create(
    system="""Ты — ассистент клиентской поддержки компании «АИ-Сервис».
    
ПРАВИЛА:
- Отвечай только про наши продукты
- Никогда не критикуй конкурентов
- Если не знаешь — предложи связаться с менеджером
- Тон: дружелюбный, профессиональный""",
    
    messages=[{
        "role": "user",
        "content": "Сколько стоит подписка?"
    }]
)
```

System prompt кэшируется у Anthropic — это быстрее и дешевле чем передавать контекст в user-сообщении.

## Антипаттерны

**Лишние правила:** Каждое правило в промпте снижает вероятность соблюдения остальных. Оставляй только важное.

**Двойное отрицание:** «Не давай неточных ответов» → «Давай только точные ответы». Позитивные инструкции работают лучше.

**Перегрузка задачами:** Один промпт — одна основная задача. Несколько задач → несколько вызовов.

**Нет примеров:** Если формат важен — дай 1-2 примера. Описание формата словами менее точное.

---

::: info Тестирование промптов
Хороший промпт стабилен: на одном и том же входе даёт похожий результат. Проверяй на 5-10 разных входах, включая граничные случаи. Сохраняй версии — промпты ломаются при обновлении моделей.
:::
