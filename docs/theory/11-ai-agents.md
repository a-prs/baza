# AI-агенты: следующий уровень автоматизации

Агент — это ИИ который не просто отвечает на вопросы, а сам планирует действия, использует инструменты и доводит задачу до результата. Понимание агентов открывает класс задач которые раньше требовали программиста.

## Чем агент отличается от чат-бота

**Чат-бот:** вопрос → ответ. Один обмен, человек интерпретирует результат.

**Агент:** задача → план → инструменты → выполнение → результат. Сам разбивает задачу на шаги, использует инструменты (поиск, код, файлы), корректирует курс при ошибках.

```
Чат-бот:
«Как найти дубликаты в CSV?» → объяснение

Агент:
«Найди дубликаты в файле data.csv» →
  → читает файл
  → анализирует структуру
  → пишет код
  → запускает
  → показывает результат: «Найдено 47 дубликатов в колонке email»
```

## Ключевые компоненты агента

**1. LLM (мозг)** — модель которая планирует, принимает решения, генерирует код.

**2. Инструменты (руки)** — функции которые агент может вызывать:
- Чтение/запись файлов
- Выполнение кода
- Поиск в интернете
- HTTP-запросы к API
- Работа с базой данных

**3. Память** — что агент помнит:
- Контекст текущей сессии (короткая)
- Файлы с заметками (долгая)
- Векторная БД для поиска по истории

**4. Цикл действий** — AgentLoop:
```
Задача → [Думать → Выбрать инструмент → Выполнить → Оценить] × N → Результат
```

## Claude Code как агент

Claude Code который ты уже используешь — это агент. Когда говоришь «добавь функцию оплаты в бот» — он сам:
- Читает нужные файлы
- Планирует что изменить
- Пишет код
- Проверяет что синтаксис корректен
- Часто запускает тесты

Именно поэтому Claude Code эффективнее чем просто «написать код в чате» — у него есть инструменты и он действует, а не только отвечает.

## Tool Use (вызов инструментов)

Современные LLM умеют вызывать функции. Ты описываешь инструмент — модель решает когда его использовать:

```python
from anthropic import Anthropic

client = Anthropic()

# Описываем инструменты
tools = [
    {
        "name": "get_weather",
        "description": "Получить текущую погоду в городе",
        "input_schema": {
            "type": "object",
            "properties": {
                "city": {"type": "string", "description": "Название города"}
            },
            "required": ["city"]
        }
    },
    {
        "name": "send_telegram",
        "description": "Отправить сообщение в Telegram",
        "input_schema": {
            "type": "object",
            "properties": {
                "message": {"type": "string"}
            },
            "required": ["message"]
        }
    }
]

# Функции-реализации
def get_weather(city: str) -> str:
    # реальный запрос к API
    return f"В {city}: +22°C, облачно"

def send_telegram(message: str) -> str:
    # реальная отправка
    return "Отправлено"

def run_agent(user_message: str):
    messages = [{"role": "user", "content": user_message}]
    
    while True:
        response = client.messages.create(
            model="claude-sonnet-4-6",
            max_tokens=1024,
            tools=tools,
            messages=messages,
        )
        
        if response.stop_reason == "end_turn":
            # Агент завершил — вернуть финальный ответ
            return response.content[0].text
        
        if response.stop_reason == "tool_use":
            # Агент вызвал инструмент
            messages.append({"role": "assistant", "content": response.content})
            
            tool_results = []
            for block in response.content:
                if block.type == "tool_use":
                    # Выполняем нужный инструмент
                    if block.name == "get_weather":
                        result = get_weather(**block.input)
                    elif block.name == "send_telegram":
                        result = send_telegram(**block.input)
                    
                    tool_results.append({
                        "type": "tool_result",
                        "tool_use_id": block.id,
                        "content": result
                    })
            
            messages.append({"role": "user", "content": tool_results})
```

## MCP (Model Context Protocol)

MCP — стандарт Anthropic для подключения инструментов к агентам. Вместо того чтобы вшивать каждый инструмент в код — MCP-сервер предоставляет их через единый интерфейс.

Ты уже использовал MCP: когда настраивал n8n MCP-сервер, Claude Code получил инструменты для работы с n8n без единой строки твоего кода.

```
Claude Code ←→ MCP-сервер (n8n) ←→ n8n workflows
Claude Code ←→ MCP-сервер (ЯД)  ←→ Яндекс.Диск
Claude Code ←→ MCP-сервер (БД)  ←→ PostgreSQL
```

Готовые MCP-серверы: [github.com/modelcontextprotocol/servers](https://github.com/modelcontextprotocol/servers)

## Практические применения агентов сейчас

**Кодинг-агент** — пишет, тестирует, исправляет код (Claude Code).

**Исследование** — находит информацию из разных источников, синтезирует отчёт.

**Обработка документов** — читает PDF/Excel, извлекает данные, формирует сводку.

**Мониторинг и реакция** — проверяет сайт, находит проблему, отправляет алерт с диагностикой.

**Генерация контента** — по теме и стилю создаёт серию постов, адаптирует под разные каналы.

## Пример: агент-помощник через n8n

В n8n можно собрать простого агента без кода:

1. **Webhook** — принимает задачу
2. **AI Agent** нода → модель + инструменты:
   - HTTP Request (вызов внешних API)
   - Code (выполнение JS)
   - Google Sheets (чтение/запись)
3. **Telegram** — отправляет результат

Агент сам решает какой инструмент использовать для выполнения задачи.

## Когда агент не нужен

Агент добавляет сложность — используй только если задача реально требует автономных действий.

**Простой скрипт достаточно:** регулярная выгрузка данных, парсинг по фиксированному алгоритму, автопостинг по расписанию.

**Агент нужен:** задача меняется каждый раз, нужно принимать решения по ходу, много разных инструментов в зависимости от ситуации.

## Тренд: от функций к агентам

Раньше: разработчик писал функцию для каждого сценария.

Сейчас: описываешь инструменты → агент сам комбинирует их для любого сценария.

Ближайшие 2–3 года — переход к «агентным системам»: вместо кнопок и форм пользователь говорит что хочет, агент делает. Вайб-кодер который понимает как это работает — на несколько шагов впереди.
