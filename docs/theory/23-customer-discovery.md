# Кастдев: разговоры с пользователями

Customer Discovery (кастдев) — главный навык строителя продукта. Правильные разговоры с пользователями дают больше чем месяц разработки. Неправильные — дают ложную уверенность.

## Почему большинство кастдевов бесполезны

Стандартные ошибки:

```
❌ «Тебе понравился бы такой бот?»
— «Да, конечно!» (люди вежливы)

❌ «Насколько ты оцениваешь эту идею?»
— «8 из 10» (не несёт информации)

❌ «Ты бы использовал это?»
— «Наверное да» (гипотетическое будущее = ненадёжно)
```

Людей нужно спрашивать о прошлом поведении, не о гипотетическом будущем.

## Правильные вопросы (принцип «Мамы-теста»

Дейн Марковитц описал «Тест Мамы»: если бы твоя мама могла солгать из вежливости — эти вопросы она не смогла бы неправильно ответить, потому что они о фактах.

```
✅ «Расскажи как ты решаешь [проблему] сейчас?»
✅ «Когда последний раз ты сталкивался с этим? Что конкретно произошло?»
✅ «Что тебя больше всего раздражает в том, как ты это делаешь сейчас?»
✅ «Сколько ты платишь за это сейчас? / Сколько времени тратишь?»
✅ «Ты пробовал другие решения? Почему перестал?»
```

## Структура 30-минутного интервью

```
0-5 мин: Контекст
— «Расскажи чем занимаешься / что за задача стоит перед тобой»
— «Как часто ты сталкиваешься с [проблемой]?»

5-20 мин: Углубление в проблему
— «Расскажи конкретный случай когда это произошло»
— «Что было самым сложным в этот момент?»
— «Что ты делал дальше?»
— «Почему именно так, а не по-другому?»

20-25 мин: Текущие решения
— «Что используешь сейчас?»
— «Сколько платишь / сколько времени уходит?»
— «Что больше всего не устраивает в текущем решении?»

25-30 мин: Завершение
— «Есть ли что-то что я не спросил, но стоит знать?»
— «Кого ещё посоветуешь поговорить на эту тему?»
```

НЕ показывай продукт до разговора о проблеме.

## Признаки что разговор дал сигнал

**Сильные сигналы (строй это):**
- «Я бы прямо сейчас заплатил за это»
- «Мы теряем из-за этого X в месяц»
- «Я уже плачу за [решение], но оно [конкретная проблема]»

**Слабые сигналы (продолжай исследовать):**
- «Интересная идея»
- «Было бы классно иметь»
- «Да, наверное пригодилось бы»

**Тревожные сигналы (останови разработку):**
- Не могут вспомнить конкретный случай с проблемой
- «Иногда бывает» без деталей
- Предлагают функции, не говорят о боли

## Как находить людей для интервью

```
B2C:
- Telegram-чаты по теме (не спам — отвечай на вопросы, знакомься)
- Ретаргетинг в соцсетях (100-200 человек ЦА)
- Форумы / Reddit / профсообщества
- Свои знакомые из ЦА

B2B:
- LinkedIn (написать напрямую)
- Конференции по нише
- Партнёры / коллаборации
- Холодные письма (конкретный повод + ценность)

Скрипт для приглашения на интервью:
«Привет! Я строю [продукт] для [ЦА]. Ищу людей кто [конкретная роль] чтобы
лучше понять как вы [задача]. Не продаю ничего — только 20 минут разговора.
Взамен [кофе / пожертвование в фонд / просто расскажу что строю].
Удобно на этой неделе?»
```

## Что делать с результатами

### Найти паттерны (3+ интервью)

Собери ответы в таблицу:

| Человек | Текущее решение | Главная боль | Готов платить | Цена |
|---|---|---|---|---|
| Алексей | Excel | Долго вручную | Да | 3000₽ |
| Мария | Ничего | Забываю следить | Нет | — |
| Сергей | Конкурент X | Нет интеграции | Да | 5000₽ |

Паттерн: 2 из 3 называют ручной труд и готовы платить → это реальная проблема.

### Обновить гипотезу

```
ДО: «Нужен бот для управления задачами»
ПОСЛЕ: «Нужен бот для фрилансеров, которые ведут несколько клиентов в WhatsApp
и тратят час в день на переключение между чатами. Хотят видеть все дедлайны в одном месте.»
```

Конкретность = меньше ошибок при разработке.

## Красные флаги в кастдеве

**«Расскажи о своём продукте»** — не делай этого в начале. Сначала проблема, потом решение.

**Защита идеи:** когда собеседник сомневается — не убеждай его. Его сомнение — ценные данные.

**Слышишь что хочешь услышать:** если все 5 человек говорят «да», перепроверь — возможно, ты задаёшь наводящие вопросы.

**Разговариваешь с нецелевой аудиторией:** если продукт для малого бизнеса — говори с малым бизнесом, не с друзьями-разработчиками.

## Минимальный кастдев (5 разговоров)

Достаточно для валидации или опровержения гипотезы:

```
1. Найти 5 человек из ЦА
2. По 20-30 минут каждый
3. Фокус: «Как ты делаешь X сейчас?» + «Что самое сложное?»
4. После 5: написать 3-5 главных инсайтов
5. Решить: строить или менять гипотезу
```

Если после 5 разговоров нет ни одного «хочу платить прямо сейчас» — гипотезу надо менять.

---

::: info Кастдев не заканчивается
Разговоры с пользователями — не разовая активность перед запуском. Лучшие строители продуктов говорят с пользователями еженедельно даже после Product-Market Fit. Это единственный способ не проспать момент когда рынок изменился.
:::
