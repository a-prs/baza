# Запуск MVP: от идеи до первых пользователей за неделю

MVP — не «продукт с минимальными функциями». MVP — самый быстрый способ проверить, хочет ли кто-то платить за твою идею. Разбираем процесс без воды.

## Неделя до запуска: что делать

### День 1: Сформулировать гипотезу

Хорошая гипотеза = конкретная. Не «сделаю AI-бота», а:

```
«[Владельцы малого бизнеса] хотят [автоматически отвечать на вопросы клиентов в WhatsApp],
чтобы [не нанимать оператора].
Готовы платить [1500₽/мес].
Я узнаю это верно если [5 человек предоплатят за доступ к бете].»
```

Зафиксируй критерий валидации ДО начала разработки.

### День 2-3: Минимальный продукт

Правило: реши задачу пользователя с наименьшим возможным кодом.

```
Идея: AI-ассистент для юристов, ищет по документам
Плохой MVP: векторная БД + fine-tuning + веб-интерфейс + авторизация + аналитика
Хороший MVP: Telegram-бот + один PDF в контекст + Claude
```

Функций ровно столько, чтобы пользователь мог решить свою задачу и заплатить.

### День 4: Оплата с первого дня

Не откладывай монетизацию. Без оплаты не знаешь реального спроса.

```
Минимальный сценарий оплаты:
1. Пользователь видит что бот платный
2. /subscribe → инвойс Stars или ссылка ЮКасса
3. Оплатил → получает доступ
4. Не оплатил → лимит (5 запросов/день)
```

### День 5: Страница с описанием

Не нужен полноценный лендинг. Достаточно:
- Один абзац что делает продукт
- Конкретный результат («Сокращает время ответа на запросы клиентов с 2 часов до 2 минут»)
- Кнопка «Попробовать» → бот
- Цена

Можно сделать в Telegra.ph за 10 минут.

### День 6-7: Первые пользователи

Не жди «пока готово» — ищи пользователей параллельно.

**Где искать первых:**
- Telegram-чаты по нише (не спам, а отвечай на вопросы + оставь ссылку)
- Свои контакты — кого лично знаешь из ЦА
- Тематические форумы / Reddit / VC.ru
- LinkedIn (для B2B)

**Первый питч (3 предложения):**
```
«Сделал бота который [делает X для Y]. Если ты [ЦА] — попробуй бесплатно 3 дня, 
потом [цена]. Вот ссылка: t.me/yourbot»
```

## Метрики первой недели

Что измерять сразу:

```python
# Минимальный трекинг в боте
@dp.message(Command("start"))
async def cmd_start(message: Message):
    user_id = message.from_user.id
    source = message.text.split()[-1] if len(message.text.split()) > 1 else "direct"
    
    await db.execute(
        "INSERT OR IGNORE INTO users (telegram_id, source, created_at) VALUES (?,?,CURRENT_TIMESTAMP)",
        (user_id, source)
    )
    # source = реферальный параметр из /start ИСТОЧНИК
```

Ключевые метрики:
- **Activation rate:** % пришедших → сделали первое ключевое действие
- **Conversion:** % активных → оплатили
- **Retention D7:** % вернулись через неделю

Для MVP достаточно вести в таблице Google Sheets вручную.

## Ошибки первой недели

### Строишь то что не просили

Признак: делаешь фичи которые «должны понравиться», а не те что просят.

Решение: разговаривай с первыми 5 пользователями каждый день. «Что тебе мешает использовать это чаще?» — и строй ответ на этот вопрос.

### Перфекционизм

Признак: «Ещё чуть-чуть допилю и запущу».

Решение: установить дедлайн (например, пятница 18:00) и запустить что есть. Несовершенный MVP который видят люди — лучше идеального продукта который никто не пробовал.

### Слишком широкий рынок

Признак: «Подходит для всех малых бизнесов».

Решение: сузить до одной ниши. «Только для стоматологических клиник» → легче найти первых клиентов, легче понять их проблемы, легче сделать продающее сообщение.

### Нет четкого CTA

Признак: людям нравится, но никто не подписывается.

Решение: один четкий призыв к действию. «Попробуй 3 дня бесплатно → /start» — не «узнай больше», не «оставь контакт».

## Валидация до разработки

Можно проверить спрос ещё до написания кода:

```
1. Создать бота с /start и форму предзаказа
2. Написать 20 людям из ЦА
3. Если 3+ готовы заплатить — строить
4. Если нет — итерировать гипотезу
```

Инструмент: Typeform / Google Form для сбора заявок. Деньги можно собрать через Stripe через «предзаказ».

## После запуска: первые 30 дней

```
Неделя 1: найти первых 10 пользователей, поговорить с каждым
Неделя 2: исправить самое большое препятствие к использованию
Неделя 3: пройти первую конверсию в платёж
Неделя 4: понять кто возвращается и почему

Цель месяца: 3-5 платящих пользователей, понять почему они платят
```

Не 1000 пользователей. 3-5 платящих дают больше информации для роста.

## Шаблон MVP за выходные

```
Пятница вечер:
□ Сформулировать гипотезу (30 мин)
□ Выбрать стек (5 мин — всегда одно и то же)
□ Создать бота, настроить бд, базовый /start (2 часа)

Суббота:
□ Основная функция (4 часа)
□ Оплата через Stars или ЮКасса (2 часа)
□ Задеплоить на VPS (1 час)

Воскресенье:
□ Страница описания (1 час)
□ Написать первым 10 потенциальным пользователям (2 часа)
□ Исправить то что сломалось после первых тестов (2 часа)

Понедельник:
□ Первый реальный пользователь
```

---

::: info Главное правило MVP
Цель MVP — научиться, не построить. Каждый день без пользователей в реальном продукте — это день без обучения. Запускай раньше чем кажется готовым.
:::
